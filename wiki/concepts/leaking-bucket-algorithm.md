# 누출 버킷 알고리즘 (Leaking Bucket Algorithm)

## 한 줄 정의

요청을 FIFO 큐에 적재하고 **일정 속도로 처리(누출)** 하는 처리율 제한 알고리즘. 토큰 버킷과 달리 처리율을 평탄화한다 (ch04, p.62).

## 왜 필요한가

다운스트림이 일정한 처리율을 전제로 한 경우(결제 게이트웨이·재고 차감 등)에는 트래픽이 **부드럽게 흘러야** 한다. 누출 버킷은 outflow를 고정시켜 이를 보장한다. Shopify가 채택 (ch04, p.62).

## 핵심 메커니즘

두 파라미터:

- **Bucket size**: 큐 길이.
- **Outflow rate**: 초당 처리 가능한 요청 수.

동작:

1. 요청 도착 → 큐가 안 차 있으면 enqueue.
2. 가득 차 있으면 drop.
3. 워커가 일정 간격으로 큐에서 꺼내 처리.

## 트레이드오프

**Pros**
- 큐 크기로 메모리 상한 명확.
- **일정 outflow rate** 보장 — 다운스트림이 안정.

**Cons**
- 버스트가 들어와 큐를 채우면 **그 안의 오래된 요청이 처리되는 동안 최근 요청이 차단**될 수 있음 → 최신 요청이 손해.
- 두 파라미터 튜닝 부담은 토큰 버킷과 같다.

## 토큰 버킷과의 비교

| 측면 | Token bucket | Leaking bucket |
|---|---|---|
| 버스트 처리 | 토큰 남는 동안 허용 | 큐 길이 만큼만 흡수, 처리는 평탄 |
| outflow | 가변 | 고정 |
| 적합한 상황 | API 게이트웨이 일반 | 다운스트림 처리율 일정 보장 필요 |
